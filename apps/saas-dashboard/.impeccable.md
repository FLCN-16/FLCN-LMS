# FLCN SaaS Dashboard — Design System

## Design Context

### Users
- **Who**: Internal SaaS operations team (super admins, billing managers)
- **Context**: Managing customer licenses, subscriptions, billing, and API credentials
- **Job**: Quickly assess platform health, manage resources, execute operations efficiently
- **Emotional tone**: Confident, in-control, trustworthy, professional

### Brand Personality
**3 words**: Efficient, Premium, Trustworthy

### Aesthetic Direction
- **Visual tone**: Modern minimalist with operational clarity
- **Primary style**: Flat Design (no shadows, clean lines, color-driven)
- **Key colors**: Gold (#F59E0B) for trust/warmth + Purple (#8B5CF6) for technology
- **Dark/Light**: Both modes equally polished
- **References**: Linear SaaS dashboards (clean, data-first) + Stripe (refined minimalism)
- **Anti-references**: Gradients, excessive shadows, decorative animations, neon colors

## Design Principles

1. **Clarity over decoration** — Remove visual noise; make operations data instantly readable
2. **Hierarchy through scale and weight** — Use Calistoga (bold headlines) + Inter (body) to create clear visual levels
3. **Color for meaning** — Each metric/action uses distinct, semantic colors (gold = licenses, violet = API, emerald = revenue)
4. **Fast interactions** — Hover states change within 150-200ms; no disruptive animations
5. **Operational intelligence** — Every dashboard card is a clickable action; visual density respects scanning patterns

### Typography

- **Display/Headings**: Calistoga (bold, warm, premium)
- **Body**: Inter (readable, modern, accessible)
- **Mono**: Courier/Geist Mono (API keys, code snippets)

**Scale**:
- H1: 36-48px (dashboard title)
- H2: 20-24px (section headers)
- H3: 16-18px (card titles)
- Body: 14-16px (default)
- Small: 12-13px (labels, hints)

### Spacing & Rhythm

- Base unit: 4px
- Component padding: 16px (4 units), 24px (6 units), 32px (8 units)
- Section gaps: 32px, 48px
- Touch target minimum: 44px × 44px

### Effects & Motion

- **Hover**: Color shift + opacity adjustment (150ms ease-out)
- **Active/Press**: Scale 0.95 with color shift (100ms)
- **Transition duration**: 150-200ms for hover, 200-300ms for state changes
- **Easing**: ease-out for enter, ease-in for exit
- **No animations** > 300ms or decorative-only

## Component Guidelines

### Stat Cards

- Display icon in top-right with semantic color
- Large metric number (48px+) with bold weight
- Label in small caps, muted color
- Entire card is clickable with hover state
- Supports loading state with skeleton

### Navigation Buttons

- Icon + text label (never icon-only)
- Gradient background on hover (amber → violet)
- Arrow indicator aligns right
- Minimum 44px height for touch
- Active state highlights primary color

### Sections

- Labeled with small-caps header + icon
- Visual separator above section (border or spacing)
- Consistent 32px+ gap between sections
- No nested cards within cards

## Accessibility Checklist

- [ ] Text contrast: 4.5:1 minimum (WCAG AA)
- [ ] Focus indicators: 2-4px ring around interactive elements
- [ ] Touch targets: 44×44px minimum
- [ ] Color not alone: Pair color with icons/text
- [ ] Keyboard navigation: Tab order matches visual order
- [ ] prefers-reduced-motion: Animations respect user setting
- [ ] Screen reader: Descriptive labels on all interactive elements
- [ ] Dark mode: Tested for contrast separately (not inverted)

## Anti-Patterns to Avoid

❌ Mixing flat + skeuomorphic styles  
❌ Using emojis as icons  
❌ Animating layout properties (width, height, top, left)  
❌ Shadows or excessive blur  
❌ Color-only meaning indicators  
❌ Cramped information density  
❌ Horizontal scroll on mobile  
❌ Decorative-only animations  
❌ Modal-heavy navigation  
❌ Gray text on gray backgrounds  

## Pages

- **Plans**: CRUD operations, pricing tiers, feature toggles
