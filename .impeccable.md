# FLCN-LMS Design System

## Design Context

### Users
**FLCN internal operations team** — a small, focused team managing:
- License issuance and lifecycle (activate, suspend, revoke)
- Subscription billing and invoices
- Customer API keys and credentials
- Super admin accounts and access control

**Context**: These are power users who interact with the dashboard daily. They need speed, clarity, and efficient workflows. They're comfortable with technical concepts but value ease of navigation.

**Job to be done**: Quickly issue licenses, manage customer subscriptions, track billing, and maintain system security.

### Brand Personality
**Modern, innovative, forward-thinking** — FLCN is a cutting-edge EdTech platform with a SaaS operations backbone. The brand should feel:
- **Contemporary**: Up-to-date with current design standards
- **Trustworthy**: Reliable, secure, professional
- **Efficient**: Gets out of the way; lets users accomplish tasks quickly

### Aesthetic Direction
**Minimal & Refined** (inspired by Linear, Vercel, Stripe)
- Clean typography with intentional hierarchy
- Generous whitespace and breathing room
- Focus on content over decoration
- Sophisticated color treatment (not trendy, timeless)
- Functional elegance—every element earns its place

**Key principle**: Modern sophistication without coldness. The interface should feel premium and considered, not minimal to the point of being stark.

### Design Principles

1. **Clarity through simplicity**
   - Remove visual noise and distraction
   - Single dominant hierarchy per page
   - Information presented in logical, digestible groups

2. **Contemporary typography as brand**
   - Distinctive font pairing: elegant display + refined body
   - Fluid sizing using clamp() for responsive hierarchy
   - Intentional letter spacing and line height for sophistication

3. **Intent over decoration**
   - No gratuitous visual effects (glassmorphism, heavy shadows, gradients)
   - Every color, spacing, and element serves a purpose
   - Refine the fundamentals (grid, type, color) rather than add complexity

4. **Performance and efficiency**
   - Fast, responsive interactions
   - Optimistic UI updates
   - Minimize cognitive load—data is presented clearly without oversimplification

5. **Refined color and contrast**
   - Modern OKLCH color space (not HSL/RGB)
   - Subtle, sophisticated palette—not neon or overly saturated
   - Both light and dark modes supported with equal care
   - Brand color integrated thoughtfully (not overused)

### Tech Stack & Constraints
- **Framework**: React 19, Vite, TypeScript
- **UI Library**: shadcn/ui (Radix + Tailwind CSS v4)
- **Colors**: OKLCH color space (modern, perceptually uniform)
- **Typography**: CSS custom properties for font family, size, weight
- **Icons**: lucide-react and HugeIcons
- **Theming**: Light and dark mode (system preference detection)
- **Accessibility**: WCAG AA, focus states, keyboard navigation

### What This Dashboard Is NOT
- ❌ Dark mode with glowing accents
- ❌ Gradient backgrounds or text
- ❌ Heavy shadows or glassmorphism
- ❌ Bright neons or oversaturated colors
- ❌ Rounded rectangles + thick colored borders everywhere
- ❌ Generic card grids (same-sized cards repeated endlessly)
- ❌ Large metrics with supporting text templates
- ❌ Modal overuse—prefer in-place editing and progressive disclosure

### What This Dashboard Intentionally Embraces
- ✅ Distinctive typography (not system fonts, not Inter)
- ✅ Clean data tables and lists with subtle interactions
- ✅ Asymmetrical, intentional layouts (not a rigid grid)
- ✅ Whitespace as design—breathing room between elements
- ✅ Refined color—sophisticated palette that feels considered
- ✅ Fast, optimistic interactions with minimal motion
- ✅ Progressive disclosure (basic first, advanced behind expandable sections)
- ✅ Empty states that guide and teach

## Implementation Notes

**Font Stack**: 
- Display (headings): Sophisticated sans-serif (recommend: DM Sans Bold or Space Grotesk)
- Body (text): Refined sans-serif (recommend: DM Sans Regular/Medium)
- Mono (code/keys): Monospace for technical content (Geist Mono)

**Color Palette** (OKLCH):
- Light mode: White background, subtle grays, one accent color (modern blue or teal)
- Dark mode: Near-black background, refined grays, elevated accent for hierarchy

**Spacing**: Varied spacing (not uniform padding everywhere) creates rhythm:
- Tight groupings for related items (0.5rem–1rem)
- Moderate spacing for sections (1.5rem–2rem)
- Generous spacing for major sections (2.5rem–3rem)

**Component Patterns**:
- Tables with hover states and actions on secondary reveal
- Buttons with clear hierarchy (primary, secondary, ghost, danger)
- Cards only where they logically group content—not everywhere
- Forms with inline validation and helpful error messages
- Modals only when necessary—prefer sheets and inline editing
