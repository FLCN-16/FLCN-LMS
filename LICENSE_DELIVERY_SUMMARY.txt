================================================================================
                          API ENDPOINTS (12 TOTAL)
================================================================================

PUBLIC ENDPOINTS (No Authentication Required):
  1. POST /api/v1/licenses/verify
     - Verify license key validity
     - Returns: validity status, features, cache TTL
     - Used by: Gin backend on startup

  2. POST /api/v1/licenses/check-feature
     - Check if specific feature is enabled
     - Returns: enabled status, feature limit
     - Used by: Gin backend before feature access

PROTECTED ENDPOINTS (Admin Authentication Required):
  3. POST /api/v1/licenses/issue
     - Create new license
     - Returns: Full license information (201 Created)

  4. GET /api/v1/licenses
     - List licenses with pagination & filtering
     - Supports: skip, take, search, status, isValid, instituteId, planId
     - Returns: Paginated list with metadata

  5. GET /api/v1/licenses/stats/summary
     - Get aggregate license statistics
     - Returns: total, active, expired, suspended, invalid counts

  6. GET /api/v1/licenses/:id
     - Get license by UUID
     - Returns: Complete license information

  7. GET /api/v1/licenses/key/:key
     - Get license by license key
     - Returns: Complete license information

  8. GET /api/v1/licenses/:key/features
     - Get all enabled features for license
     - Returns: Array of feature configurations

  9. PUT /api/v1/licenses/:id
     - Update license (partial update)
     - Returns: Updated license information

 10. PATCH /api/v1/licenses/:id/suspend
     - Temporarily disable license
     - Returns: Updated license with suspended status

 11. PATCH /api/v1/licenses/:id/reactivate
     - Restore suspended license
     - Returns: Updated license with active status

 12. DELETE /api/v1/licenses/:id
     - Permanently revoke license
     - Returns: Revocation confirmation

================================================================================
                        DATABASE SCHEMA
================================================================================

Table: licenses (Master Database)

Columns:
  • id (UUID, Primary Key)
  • license_key (VARCHAR, Unique) - The actual license key
  • organization_name (VARCHAR) - Organization using the license
  • status (ENUM) - active | expired | invalid | suspended | pending
  • is_valid (BOOLEAN) - Quick validity check
  • expiry_date (TIMESTAMP, nullable) - When license expires
  • features (JSONB) - Array of {name, enabled, limit?}
  • max_users (BIGINT) - Maximum users allowed (0 = unlimited)
  • plan_id (UUID, nullable) - Associated plan
  • institute_id (UUID, nullable) - Associated institute
  • issued_by_id (UUID, nullable) - Admin who issued license
  • notes (TEXT, nullable) - Internal documentation
  • last_verified_at (TIMESTAMP, nullable) - Last verification time
  • verification_count (BIGINT) - Number of verifications
  • created_at (TIMESTAMP) - Creation timestamp
  • updated_at (TIMESTAMP) - Last update timestamp

Indexes:
  • UNIQUE idx_licenses_key (license_key)
  • idx_licenses_status (status)
  • idx_licenses_institute_id (institute_id)
  • idx_licenses_plan_id (plan_id)
  • idx_licenses_is_valid (is_valid)
  • idx_licenses_expiry_date (expiry_date)

================================================================================
                      INTEGRATION WORKFLOW
================================================================================

Gin Backend Integration:

  1. STARTUP
     ├─ Load cached license from database
     ├─ Call POST /api/v1/licenses/verify
     ├─ Cache result with TTL from response
     └─ Start background verification cron job

  2. BEFORE FEATURE ACCESS
     ├─ Call POST /api/v1/licenses/check-feature
     ├─ Check "enabled" flag in response
     └─ Allow/deny feature access

  3. NETWORK FAILURE
     ├─ Use cached license if available
     ├─ Log failure but continue operation
     └─ Retry on next scheduled verification

NestJS Dashboard Integration:

  1. LICENSE CREATION
     ├─ Call POST /api/v1/licenses/issue (protected)
     ├─ Provide JWT bearer token
     └─ Create license with features and expiry

  2. LICENSE MANAGEMENT
     ├─ List licenses: GET /api/v1/licenses (with filters)
     ├─ Update licenses: PUT /api/v1/licenses/:id
     ├─ Suspend/reactivate: PATCH /api/v1/licenses/:id/suspend
     └─ Revoke licenses: DELETE /api/v1/licenses/:id

  3. MONITORING
     ├─ View statistics: GET /api/v1/licenses/stats/summary
     ├─ Search and filter licenses
     └─ Track verification counts
